# OPTOStokes Optical Filter Resources

Project Preview Address: https://snazzy-salamander-3c1a10.netlify.app/

This repository contains a lightweight public resource page for optical filter selection. It is designed for GitHub Pages and points readers to the complete OPTOStokes catalog at:

https://www.optofilters.com/optical-filters-catalog.html

## Purpose

- Provide a quick, readable overview of optical filter categories.
- Share selected model examples without publishing the full product database.
- Help engineers and buyers continue to the official OPTOStokes website for complete tables, specifications, and quotation support.

## What This Repository Includes

- `index.html`: GitHub Pages resource page with application paths, a spectrum browsing map, selected model examples, and catalog links.
- `assets/styles.css`: local CSS, no external dependency.
- `data/featured-products.json`: small public data sample used as a reference source.
- `compliance-checklist.md`: public repository and GitHub Pages review checklist.
- `link-map.md`: official OPTOStokes destination links.

## What This Repository Does Not Include

- No private customer information.
- No pricing database.
- No account, payment, login, or file upload flow.
- No hidden redirect, short link, or tracking script.
- No full copy of the main OPTOStokes catalog.

## What Not To Upload

- ZIP archives from local experiments
- OS metadata such as `.DS_Store`, `Thumbs.db`, and `desktop.ini`
- editor folders such as `.idea/` and `.vscode/`
- temporary files such as `*.tmp`, `*.bak`, `*.log`, and `*.swp`
- dependency caches such as `node_modules/` if they are ever added locally

## Content Strategy

The page is designed to create browsing interest before sending readers to the main OPTOStokes website:

- Start by application, not by a long product list.
- Show only representative models.
- Explain what engineers should check next.
- Link to the full catalog only when the surrounding context is useful.

## Recommended GitHub Pages Setup

1. Create a public GitHub repository.
2. Upload the contents of this `V3` folder to the repository root.
3. In GitHub, open `Settings -> Pages`.
4. Choose `Deploy from a branch`.
5. Select the default branch and root folder.
6. Use the generated GitHub Pages URL as a public resource link.
